Notes

 

Nova's resources all start at ID number 128, but the internal storage for all data file info is zero-based. Therefore, when a field in the Nova data file is said to refer to a government, stellar object, etc., it refers to it by its index number (starts at 0) unless it is specifically stated that it is referring to the ID number, which starts at 128.

Some of Nova's fields refer to other resource IDs or index numbers, but their values are offset by a certain amount to indicate type. For example, the mïsn resource's AvailStel field refers to the index number of a gövt resource when its value is between 10000 and 10255. In cases like this, it is necessary to add to or subtract from the field in order to force the value into the proper range: in this instance, you'd subtract 10000 to find the index number of the gövt.

Any resources in an Nova plugin file automatically replace same-numbered resources in Nova's main files. Resources are loaded from the "Nova Files" folder first and then resources from the "Nova Plug-Ins" folder are loaded.
